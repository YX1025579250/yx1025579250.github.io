JAVA排序算法小结

发表于 2019-12-17 更新于 2019-12-18 分类于 JAVA
本文字数： 7.5k 阅读时长 ≈ 7 分钟

前言

​ 今天在牛客网上刷题时，做到一道关于排序的题，想想自己似乎还没有好好学习，整理归纳过排序算法。于是，趁着现在有空，好好整理归纳一下经典的排序算法，并用JAVA去实现一下。这里仅仅进行排序算法的JAVA实现，排序算法的相关理论请自行Google。

冒泡排序

最佳情况：T(n) = O(n) 最差情况：T(n) = O(n2) 平均情况：T(n) = O(n2)

选择排序

最佳情况：T(n) = O(n2) 最差情况：T(n) = O(n2) 平均情况：T(n) = O(n2)

插入排序

最佳情况：T(n) = O(n) 最坏情况：T(n) = O(n2) 平均情况：T(n) = O(n2)

希尔排序

希尔排序的复杂度取决于步长序列的选择，但总的来说肯定优于前三种排序，甚至在数据量较小时，优于快速排序和堆排序

最佳情况：T(n) = O(nlogn​) 最差情况：T(n) = O(nlogn) 平均情况：T(n) = O(nlogn)

快速排序

最佳情况：T(n) = O(nlogn) 最差情况：T(n) = O(n2) 平均情况：T(n) = O(nlogn)　

堆排序

最佳情况：T(n) = O(nlogn​) 最差情况：T(n) = O(nlogn) 平均情况：T(n) = O(nlogn)

计数排序

最佳情况：T(n) = O(n+k) 最差情况：T(n) = O(n+k) 平均情况：T(n) = O(n+k)

桶排序

最佳情况：T(n) = O(n+k) 最差情况：T(n) = O(n2) 平均情况：T(n) = O(n+k)

基数排序

最佳情况：T(n) = O(n * k) 最差情况：T(n) = O(n * k) 平均情况：T(n) = O(n * k)

总结

​ 原本以为小结一下排序算法不会花多少时间，但实际上却花了我整整一天的时间。前三种排序算法还算熟悉，一直以来也有用过，之后的就非常陌生了，花了不少时间去学习理解的，但是感觉还是没有理解透彻，不能滚瓜烂熟的手写出来。之后再不断复习吧，在此，特别感谢这位大佬的博客以及维基百科的大力支持。
